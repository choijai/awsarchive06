## 🔧 배포 전 필수 작업

### Step 1️⃣: Google Search Console 등록
```
1. Google Search Console 접속
   https://search.google.com/search-console

2. 속성 추가
   URL: https://awsarchive.com

3. 소유권 확인
   HTML 파일 업로드 또는 메타 태그 추가

4. Sitemap 제출
   https://awsarchive.com/sitemap.xml

5. 색인 생성 요청
   Search Console > 색인 생성 > URL 검사
```

### Step 2️⃣: Google Analytics 설정
```
1. GA4 속성 생성
2. 측정 ID 추가 (index.html의 Google Analytics 스크립트)
3. 다음 이벤트 추적:
   - 문제 생성 (문제 난이도별)
   - 프리미엄 구독 (변환 추적)
   - 언어 전환
   - 문제 제출
```

### Step 3️⃣: Bing Webmaster Tools 등록
```
1. Bing Webmaster Tools 접속
   https://www.bing.com/webmasters

2. 사이트 추가: https://awsarchive.com

3. Sitemap 제출

4. 모바일 호환성 검증
```

### Step 4️⃣: Cloudflare/DNS 설정 확인
```
1. DNS A 레코드 설정
   Domain: awsarchive.com → 서버 IP

2. www 리다이렉트 설정
   https://www.awsarchive.com → https://awsarchive.com

3. SSL 인증서 설정 (HTTPS 필수)

4. 캐시 설정 (성능 최적화)
```

## 💡 주의사항

### ❌ 피해야 할 것
```
❌ 검은 모자 SEO (Black Hat)
   - 자동 백링크 구매
   - 키워드 스터핑
   - Cloaking

❌ 도메인 변경 오류
   - 301 리다이렉트 설정 필수 (이전 도메인)
   - Search Console에서 주소 변경 등록
```

### ✅ 권장사항
```
✅ 하얀 모자 SEO (White Hat)
   - 고품질 콘텐츠 작성
   - 자연스러운 백링크
   - 사용자 경험 우선

✅ 정기적 검토
   - 주 1회 Search Console 확인
   - 월 1회 분석 리포트 작성
   - 분기별 전략 수정
```
